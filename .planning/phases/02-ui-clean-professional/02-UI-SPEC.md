## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside buttons (`gap-2` = 8px in Tailwind; use 4px for icon-internal gaps) |
| sm | 8px | Toolbar inner padding horizontal (`px-2`), separator margin (`margin: '0 4px'`) |
| md | 16px | Banner horizontal padding (`padding: '6px 16px'`), header section padding |
| lg | 24px | Main canvas vertical padding (`py-6`) |
| xl | 32px | — reserved |
| 2xl | 48px | — reserved |
| 3xl | 64px | — reserved |

Exceptions:
- Toolbar wrapper padding: `4px 8px` (xs vertical, sm horizontal) — tighter than standard `md` to keep toolbar compact
- Banner vertical padding: `6px` — between xs and sm; intentional compact treatment for the info strip
- Touch targets: not applicable (desktop-only tool; no mobile requirement per REQUIREMENTS.md Out of Scope)

*Source: CONTEXT.md + RESEARCH.md Pattern 4 (toolbar) and Pattern 5 (banner)*

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Node inputs, panel text, snapshot list items |
| Label | 12px | 500 (medium) | 1.4 | Toolbar button text (`text-sm` = 14px in Tailwind; banner text at 12px) |
| Heading | 20px | 700 (bold) | 1.2 | App header `h1` (currently `text-3xl` — retain Tailwind class; visual hierarchy unchanged) |
| Display | 30px | 700 (bold) | 1.2 | App header only (`text-3xl` = 30px) |

Precision rules:
- Toolbar buttons: `text-sm` (14px) at weight 400 — rendered via Tailwind on `Button.tsx`
- CP banner: 12px at weight 500 — via inline `fontSize: 12, fontWeight: 500`
- Node task labels: 14px at weight 400 — existing inline style in TaskNode.tsx (retain)
- App `h1`: Tailwind `text-3xl font-bold` (30px / 700) — retain existing class after removing `<big>` wrapper

*Source: RESEARCH.md Standard Stack section; CONTEXT.md button style `text-sm`; existing App.tsx Tailwind classes*

## Color

All color values are defined as named constants in `web/src/graph/theme.ts`. No hardcoded hex values may remain in component files after Phase 2.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `COLOR_BG` | `#ffffff` | Canvas background, normal node background, page background |
| Secondary (30%) | `COLOR_SURFACE` | `#f4f4f5` | Toolbar wrapper, panels, Start/End node backgrounds |
| Accent (10%) | `COLOR_ACCENT` | `#2563eb` | CP node border, focus-visible rings, CP banner border, separator — reserved list below |
| Accent light | `COLOR_ACCENT_LIGHT` | `#dbeafe` | CP node background fill only |
| Border | `COLOR_BORDER` | `#d4d4d8` | Node borders (non-CP), input borders, toolbar wrapper border, toolbar separator |
| Text primary | `COLOR_TEXT` | `#18181b` | All primary text |
| Text muted | `COLOR_TEXT_MUTED` | `#71717a` | Secondary/helper text |
| CP banner bg | — | `#eff6ff` | CP banner background only (blue-50 — distinct from `COLOR_ACCENT_LIGHT` `#dbeafe`) |
| CP banner text | — | `#1d4ed8` | CP banner text only (blue-700) |
| Destructive | — | `#ef4444` | Error-state node border (already in `styledNodes` in GraphCanvas.tsx — retain as-is) |

**Accent reserved for:**
1. CP node border: `2px solid COLOR_ACCENT` on TaskNode when `data.computed?.critical`
2. `focus-visible` outline on all interactive elements: `outline: 2px solid COLOR_ACCENT` (via Tailwind `focus-visible:outline-blue-600`)
3. CP banner border: `1px solid COLOR_ACCENT`

**Accent is NOT used for:**
- Normal node borders (use `COLOR_BORDER`)
- Toolbar wrapper border (use `COLOR_BORDER`)
- Hover states (use `COLOR_SURFACE` / `#e4e4e7`)

**Start/End node backgrounds:** `COLOR_BG` (`#ffffff`) — removes incorrect `CRITICAL_BG` usage; differentiates structural nodes via shape and label, not color. (Source: RESEARCH.md Pitfall 2 + Claude's Discretion from CONTEXT.md)

**Backward-compatible aliases (in theme.ts):**
- `CRITICAL_BG = COLOR_ACCENT_LIGHT` — alias retained for Phase 1 code compatibility
- `CRITICAL_BORDER = COLOR_ACCENT` — new alias for TaskNode border

**Ghost button states:**
- Default: no background, no border
- Hover: `background: #f4f4f5` (`COLOR_SURFACE`)
- Active: `background: #e4e4e7`
- Focus-visible: `outline: 2px solid COLOR_ACCENT, outline-offset: 2px`
- Disabled: `opacity: 0.5, cursor: not-allowed`

*Source: CONTEXT.md locked token set (verbatim) + RESEARCH.md Pattern 2 (ghost variant)*

## Anti-Patterns (Executor Must Avoid)

1. **Do not create a ToolbarButton component.** Use `Button` with `ghost` variant.
2. **Do not add CSS custom properties / CSS variables.** Token format is TypeScript constants only.
3. **Do not migrate all inline styles to Tailwind.** Only replace hardcoded hex values with token constants.
4. **Do not import `Banner.tsx` for the CP banner.** Restyle the existing inline `<div>`.
5. **Do not show a full loading overlay for PNG export.** In-button state only.
6. **Do not leave `StartNode` or `EndNode` using `CRITICAL_BG` for background.** Replace with `COLOR_BG`.
7. **Do not use `focus:` (triggers on mouse click) for ghost variant.** Use `focus-visible:` only.
